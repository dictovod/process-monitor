# 🖥 Process Monitor Pro

Telegram-бот для мониторинга новых процессов на Linux-сервере.  
Как только на сервере запускается новый процесс — бот сразу присылает уведомление в Telegram.

---

## ⚡ Быстрый старт

### 1. Залей файл на сервер

```bash
scp monitor.py root@ВАШ_IP:/root/Desktop/process-monitor/monitor.py
```

### 2. Вставь свой Telegram-токен

Открой `monitor.py` и в самом начале замени токен:

```python
TELEGRAM_TOKEN = "сюда_вставь_свой_токен"
```

> Токен получить у [@BotFather](https://t.me/BotFather) → `/newbot`

### 3. Запусти сервис

```bash
systemctl daemon-reload
systemctl enable process-monitor
systemctl start process-monitor
```

### 4. Открой бота в Telegram и напиши `/start`

## 📱 Команды

| Команда | Что делает |
|---|---|
| `/start` | Активировать бота |
| `/stop` | Отключить уведомления |
| `/status` | Показать CPU, RAM, диск, открытые порты |
| `/list` | Список игнорируемых процессов |
| `/whitelist` | Белый список процессов |
| `/settings` | Все настройки |
| `/history python3` | История запусков процесса |
| `/setcpu 5` | Не уведомлять если CPU < 5% |
| `/setram 100` | Не уведомлять если RAM < 100 MB |
| `/quiet 22:00-08:00` | Тишина ночью |
| `/quiet off` | Отключить тихие часы |

---

## 🔔 Как работают уведомления

Когда на сервере появляется новый процесс, бот присылает сообщение:

```
🔔 Новый процесс
📋 Название: curl
🆔 PID: 12345
👤 Пользователь: root
⚙️ CPU: 0.1%   💾 RAM: 4.2 MB
```

Под каждым уведомлением — кнопки:
- **🚫 Игнорировать** — этот процесс больше не будет беспокоить
- **⭐ В белый список** — важный процесс, всегда уведомлять
- **📊 Статистика** — как часто он запускался

---

## ⚙️ Режимы фильтрации

Переключаются в `/settings` → **Режим**:

| Режим | Поведение |
|---|---|
| 🚫 Чёрный список | Уведомлять обо всём, кроме игнорируемых |
| ⭐ Белый список | Только процессы из белого списка |
| 🧠 Умный | Белый список в приоритете, остальные фильтруются |

---

## 🗂 Файлы на сервере

Все данные хранятся в `/root/Desktop/process-monitor/`:

| Файл | Что хранит |
|---|---|
| `monitor.py` | Сам бот |
| `ignored_processes.json` | Игнорируемые процессы |
| `whitelist.json` | Белый список |
| `active_users.json` | Кто подключён |
| `user_settings.json` | Настройки |
| `stats.json` | Статистика запусков |
| `monitor.log` | Лог работы бота |

## 🔍 Полезные команды на сервере

```bash
# Статус сервиса
systemctl status process-monitor

# Смотреть логи в реальном времени
tail -f /root/Desktop/process-monitor/monitor.log

# Перезапустить после изменений
systemctl restart process-monitor

# Остановить
systemctl stop process-monitor
```

## ❓ Частые вопросы

**Бот молчит после `/start`**  
→ Проверь токен в `monitor.py`, перезапусти сервис

**Слишком много уведомлений**  
→ Включи группировку в `/settings`, или подними пороги `/setcpu 5` и `/setram 50`

**Хочу уведомления только о конкретных программах**  
→ Переключи режим на ⭐ Белый список и добавляй процессы кнопкой в уведомлении
